# セキュリティガイドライン

## 1. 秘密鍵の管理

### 1.1 秘密鍵の取り扱い

- **秘密鍵は絶対に他人に共有しないでください**
- 秘密鍵をメールやチャットで送信しないでください
- スクリーンショットに秘密鍵が写らないよう注意してください
- 秘密鍵のバックアップは暗号化されたオフライン媒体に保管してください

### 1.2 アプリ内での秘密鍵保護

Easy Token Mintでは以下のセキュリティ対策を実施しています：

- AES-256-GCM暗号化による秘密鍵の保護
- PBKDF2-SHA256による鍵導出（600,000イテレーション）
- メモリ上の秘密鍵は使用後即座にゼロクリア
- PINによる二重の保護

## 2. PINの設定

### 2.1 強固なPINの作成

- **8文字以上**を使用してください
- **英字と数字の両方**を含めてください
- 連続した文字（例：1234, abcd）は避けてください
- 辞書に載っている単語は避けてください
- 個人情報（誕生日、電話番号など）は使用しないでください

### 2.2 PINの管理

- PINを他人に教えないでください
- PINをメモに書いて放置しないでください
- 定期的にPINを変更することを推奨します

## 3. デバイスのセキュリティ

### 3.1 OSのセキュリティ

- OSを常に最新の状態に保ってください
- セキュリティパッチは速やかに適用してください
- ファイアウォールを有効にしてください
- 信頼できるアンチウイルスソフトを使用してください

### 3.2 物理的なセキュリティ

- デバイスを離れる際はロックしてください
- 共有PCでの使用は避けてください
- 公共のWi-Fiでの使用時は特に注意してください

## 4. トランザクションの確認

### 4.1 発行前の確認事項

発行（ミント）を実行する前に、必ず以下を確認してください：

1. **発行先アドレス**が正しいか
2. **発行量**が意図した数量か
3. **トークン**が正しいか
4. **ガス代**が妥当な範囲か

### 4.2 高額発行時の注意

大量のトークンを発行する際は：

- 二重チェックを行ってください
- 可能であれば少量でテスト発行を行ってください
- 複数人で確認することを推奨します

## 5. バックアップ

### 5.1 定期的なバックアップ

以下のデータを定期的にバックアップしてください：

- 秘密鍵（暗号化して安全な場所に保管）
- データベースファイル（自動バックアップされますが、外部保存も推奨）

### 5.2 バックアップの保管

- 複数の場所にバックアップを保管してください
- オフライン媒体への保管を推奨します
- バックアップも暗号化してください

## 6. 緊急時の対応

### 6.1 秘密鍵が漏洩した場合

1. 直ちに全てのトークンを安全なウォレットに移動
2. 該当アドレスのMINTER_ROLEを即座に取り消し
3. 新しいウォレットで権限を再設定

### 6.2 不正アクセスが疑われる場合

1. ネットワークからデバイスを切断
2. 全てのアクティブなセッションを終了
3. PINを変更
4. 操作ログを確認し、不正な操作がないか確認

## 7. 推奨事項

- [ ] 専用のデバイスでの運用を推奨
- [ ] 定期的なセキュリティ監査の実施
- [ ] 操作ログの定期的な確認
- [ ] 使用していないMINTER_ROLEの取り消し
- [ ] Admin権限の冗長化（複数アドレスへの付与）
