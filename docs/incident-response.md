# インシデント対応手順

## 1. 概要

本ドキュメントでは、Easy Token Mintの運用中に発生する可能性のあるセキュリティインシデントへの対応手順を定義します。

## 2. インシデント分類

### 2.1 重大度レベル

| レベル | 説明 | 対応時間 |
|--------|------|----------|
| Critical | 秘密鍵漏洩、不正発行発生 | 即時対応 |
| High | 不正アクセス疑い、システム障害 | 1時間以内 |
| Medium | 異常な操作検知、パフォーマンス低下 | 24時間以内 |
| Low | 軽微なバグ、UIの問題 | 次回アップデートで対応 |

## 3. インシデント種別と対応

### 3.1 秘密鍵の漏洩・紛失

#### 検知のサイン
- 不審なトランザクションの発生
- 身に覚えのない権限変更
- 秘密鍵を含むファイルの流出

#### 即時対応（Critical）

1. **ネットワーク切断**
   - 被害拡大防止のため、該当デバイスをオフラインに

2. **権限の取り消し**
   ```
   - バックアップのAdmin権限を使用
   - 漏洩したアドレスのMINTER_ROLEを即座に取り消し
   - 必要に応じてAdmin権限も取り消し
   ```

3. **資産の移動**
   - 該当ウォレットの全資産を安全なアドレスに移動

4. **証拠の保全**
   - 操作ログのエクスポート
   - ブロックチェーン上のトランザクション記録

5. **新しい運用体制の構築**
   - 新しいウォレットの作成
   - 権限の再設定

#### 事後対応
- 原因分析の実施
- 再発防止策の策定
- 関係者への報告

### 3.2 不正な発行の検知

#### 検知のサイン
- 予定にない大量発行
- 未知のアドレスへの発行
- 異常な時間帯の発行

#### 対応手順（Critical/High）

1. **状況の確認**
   - 履歴画面で発行記録を確認
   - 発行先アドレスの特定
   - 発行量の把握

2. **発行の停止**
   - 該当トークンのMINTER_ROLEを全て取り消し
   - 必要最小限の権限のみ再付与

3. **影響範囲の特定**
   - 不正に発行されたトークン量
   - 流通状況の確認

4. **対応策の検討**
   - トークンの追加発行による希釈化防止
   - 法的対応の検討（必要に応じて）

### 3.3 システム障害

#### 検知のサイン
- アプリが起動しない
- トランザクションが送信できない
- データベースエラー

#### 対応手順（High/Medium）

1. **エラーログの確認**
   - ログフォルダを開く
   - エラーメッセージを特定

2. **基本的なトラブルシューティング**
   - アプリの再起動
   - ネットワーク接続の確認
   - RPC接続の確認

3. **データの復旧**
   - 自動バックアップからの復元
   - 手動バックアップからの復元

4. **エスカレーション**
   - 解決しない場合は開発チームに報告

### 3.4 PIN認証の問題

#### 検知のサイン
- PINを忘れた
- 不正なPIN試行によるロック

#### 対応手順

1. **ロック状態の確認**
   - ロック解除まで待機（最大30分）

2. **PIN忘れの場合**
   - 秘密鍵のバックアップから復元
   - アプリを再インストール
   - 秘密鍵を再インポート

## 4. 連絡体制

### 4.1 連絡先リスト

| 役割 | 担当 | 連絡先 |
|------|------|--------|
| 運用責任者 | - | - |
| 技術責任者 | - | - |
| セキュリティ責任者 | - | - |

### 4.2 エスカレーション

1. 初動対応者 → 運用責任者
2. 運用責任者 → 技術責任者/セキュリティ責任者
3. 必要に応じて → 経営層/法務

## 5. 事後対応

### 5.1 インシデントレポート

以下の項目を含むレポートを作成：

- 発生日時
- 検知方法
- 影響範囲
- 対応内容
- 原因分析
- 再発防止策

### 5.2 再発防止

- セキュリティ設定の見直し
- 運用手順の改善
- 監視体制の強化
- 担当者への教育

## 6. 定期訓練

### 6.1 訓練項目

- インシデント対応訓練（年2回）
- バックアップ復元テスト（年1回）
- 権限取り消し手順の確認（四半期ごと）

### 6.2 訓練記録

訓練の実施日、参加者、結果を記録し保管
